# Together — инструкция по запуску

Десктопное приложение для пары: видно, чем сейчас занят партнёр (какое
приложение открыто, активен/AFK), есть общая статистика активности,
встроенный чат и трей-иконка. Работает только на Windows (используются
нативные WinAPI-вызовы для чтения активного окна, автозапуска, трея и
т.д.) через прямое P2P-соединение по сокету между двумя ПК — без
сервера-посредника.

## 1. Установка зависимостей

Нужен Python 3.12 (или близкий 3.x — писалось и собиралось под 3.12).

Открой командную строку в папке проекта и выполни:

```
pip install -r requirements.txt
```

## 2. Настройка Zerotier (делается один раз)

Приложение стучится к партнёру напрямую по IP на порт **39721** — оба
устройства должны быть в одной приватной сети (VPN), иначе друг друга
не увидят.

1. Оба скачиваете и устанавливаете Zerotier: https://www.zerotier.com/download/
2. Один из вас регистрируется на https://my.zerotier.com и создаёт сеть (кнопка "Create A Network")
3. Копируете Network ID (16 символов)
4. Оба в трее Zerotier → "Join Network" → вставляете Network ID
5. На сайте my.zerotier.com в разделе Members — авторизуете оба устройства (ставите галочку)
6. После этого у каждого будет IP вида 10.147.X.X — это и есть адреса друг друга

Вместо Zerotier можно использовать RadminVPN или любой другой VPN, где
вы оба состоите — принцип такой же, важен только сам факт общей
приватной сети.

## 3. Первый запуск

```
python main.py
```

Приложение появится в трее (правый нижний угол, стрелочка "показать
скрытые значки"). Клик по значку или пункт меню "Открыть Together"
открывает окно.

Повторный запуск, пока предыдущий процесс уже висит в трее, блокируется
(single-instance lock) — вместо второй копии просто всплывёт
предупреждение, что приложение уже запущено.

## 4. Настройка

В окне приложения → вкладка ⚙ "Настройки":

- **Моя аватарка** — любая картинка (JPG/PNG), сама обрежется по центру в квадрат
- **Моё имя**
- **Мой пол** — влияет только на цвет карточки (своей и у партнёра), на работу не влияет
- **IP партнёра (Zerotier)** — адрес партнёра из шага 2, вида `10.147.X.X`
- **Мой Zerotier IP** — определяется автоматически, руками трогать не нужно (только если автоопределение ошиблось)
- **Ключ подключения** — общий "пароль" между вашими приложениями; должен быть **одинаковым** у обоих, иначе пакеты друг друга будут отклоняться
- **OpenRouter API ключ** *(необязательно)* — для авто-определения категории новых программ через openrouter.ai; без ключа неизвестные программы попадают в категорию "другое"
- **Игнорировать процессы** — список через запятую, которые не учитывать в статистике/активности
- **Приватный режим** — скрывает заголовки окон браузера (на случай, если за компом кто-то ещё)
- **Автозапуск с Windows**

Партнёр делает то же самое — вводит твой IP и тот же ключ подключения.
Жми "Сохранить".

Внизу вкладки настроек — два отладочных инструмента: "Что видит сканер
сейчас?" (список открытых окон, как их видит статистика) и "Лог сети"
(последние строки network_debug.log) — полезны, если соединение с
партнёром не устанавливается.

## 5. Где хранятся данные

- **Запуск из исходников** (`python main.py`) — все файлы (settings.json,
  stats.db, логи, кеш категорий) лежат прямо в папке проекта, как раньше.
- **Собранный .exe** — те же файлы переезжают в `%APPDATA%\Together`,
  чтобы не слетали при каждом перезапуске (PyInstaller `--onefile`
  распаковывает программу во временную папку заново при каждом старте).

## 6. Сборка в .exe (опционально)

```
pyinstaller --onefile --windowed --name Together --icon=assets/icon.ico main.py
```

Готовый `Together.exe` будет в папке `dist/`. Не забудь положить рядом
(или добавить через `--add-data`) папку `sound/` и `assets/icon.png` —
они читаются как ресурсы (см. `core/paths.py: resource_path`).

## Структура проекта

```
together v5/
├── main.py                  # Точка входа, single-instance lock, запуск потоков
├── requirements.txt         # Зависимости
├── settings.json            # Настройки пользователя — создаётся автоматически
├── settings.default.json    # Дефолтные значения настроек
├── stats.db                 # SQLite: история сессий + чат
├── assets/
│   ├── icon.png              # Иконка трея (рантайм)
│   └── icon.ico               # Иконка .exe (для PyInstaller)
├── sound/
│   └── alarm.wav             # Звук уведомления о новом сообщении в чате
├── core/
│   ├── tracker.py            # Чтение активного окна + AFK по бездействию клавы/мыши
│   ├── stats.py               # Статистика: сканирование всех открытых окон, топ приложений
│   ├── network.py             # P2P-связь между ПК по сокету (порт 39721)
│   ├── chat.py                 # История чата в SQLite
│   ├── tray.py                  # Иконка в трее и её меню
│   ├── autostart.py              # Автозапуск с Windows (реестр, HKCU)
│   ├── avatar.py                  # Обрезка/кодирование аватарки в base64
│   ├── ai_classify.py              # Автокатегоризация приложений через OpenRouter
│   ├── emoji_panel.py               # Открытие системной панели эмодзи Windows (Win + .)
│   ├── notifications.py              # Звук уведомления о сообщении (winsound)
│   ├── app_maps.py                    # Общий словарь "процесс → имя/категория"
│   └── paths.py                        # Пути к данным (dev vs собранный .exe)
└── ui/
    ├── window.py             # Главное окно (pywebview) + JS API + окно чата
    ├── html.py / styles.py / script.py            # Разметка/стили/логика главного окна
    └── chat_html.py / chat_styles.py / chat_script.py  # Разметка/стили/логика окна чата
```

> `core/beautiful_notifications.py` в проекте есть, но нигде не
> импортируется — старый вариант уведомлений через tkinter, оставлен на
> случай если понадобится вернуться к нему, реально не используется.
